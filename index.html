thevoicebook.me

Voicebook was created at the Facebook Hackathon Finals 2015


Honours: Finalist @ Facebook Global Hackathon Finals: Menlo Park, California

Did you know that in 2013 in the US alone, 3,154 people were killed, and an additional 424,000 were injured as a result of distracted driving? This is why we created voicebook, which strives to eliminate the pain point of distracted driving. In all essence, Voicebook is your everyday hit radio station. However, at second glance, it's much more than that.

What is the project? Voicebook is an integrated radio + social media feed station which simply plays your latest hits when your social feeds are offline, but it transforms into an automated social media aggregator/reader when your social profiles begin to receive new Facebook posts, tweets, and Instagram uploads. In this even, Voicebook will read (out loud) your social interactions in real time as they happen. This text-to-speech feature isn't limited to text and will actually analyze and verbally interpret photos for the user. Finally, an intelligent voice recognition system enables users to act on these social updates (such as commenting or liking) using their voice, without ever taking their eyes off the road.

Built with: Node.JS, Javascript, Html, CSS, Facebook Api, Clarifai Api | Website

queryhacks.herokuapp.com

Query was built at the HackingEDU Hackathon in San Mateo, CA.

Honours: Clusterpoint API Prize ($5000) @ HackingEDU: San Mateo, California

Learning has always been inspired by the active participation of all individuals. Inputs made to Query are voted on: The questions that accumulate in our app can be voted on for significance and importance to learning. Whether the raised comment or concern is something the lecturer should clarify, re-iterate, or remember for a following lecturer, participants can get a sense of what questions are being asked by peers and colleagues. Our app is dynamic and has the potential to be applied in large academic, professional, and political venues.

What is the project? Query is a real-time classroom collaboration tool designed for the university lecture. Using a unique joining code, professors can create a virtual classroom where students can ask questions and also up/downvote their peers questions. Once a student asks a questions, all other students on the platform are prompted by a modal, asking if the question is a good one or not. If this question meets a certain threshold of "yes" responses, it is displayed on the lecturer's projector screen, which he can choose to address then/at a later time. These questions can be answered through laptop, phone, or through a pebble application. As these questions are submitted, a list of curated questions are compiled on the professor's dashboard.

Built with: Node.JS, AngularJS, Socket.io, SASS, Pebble Api, Clusterpoint Api | Website, Devpost





identifai.me

IdentifaiMe was built at the DubHacks Hackathon at the University of Washington.

Honours: Best Overall/Best Facebook API @ DubHacks: Seattle, Washington

What is the hack: Identifai.Me is a program that analyzes a user’s latest photo uploads on Facebook to generate insight into the user’s recent behaviour, habits, or trends. Using both Facebook’s Development API and Clarifai’s API (where we got the inspiration for our name), Identifai.Me makes it easy for users to seamlessely derive insighful social data at the click of one button. The simple one-step authentication method, coupled with the streamlined user interface promotes widespread use and adoption.

Aside: Our team had to pivot to a different project halfway through the Hackathon due to a technical issue. Identfai’s core feature (of generating data through Facebook/Clarifai and leveraging the tagged data into sentence outputs) was conceptualized, designed, developed and deployed in a span of 7 hours.

Built with: Node.JS, ejs, CSS, Facebook Api, Clarifai Api | Website, Devpost

Restaurant Quiz

RestaurantQuiz is an Android game built for CPSC 210.

What is the project? RestaurantQuiz is an Android Game made as a course project for my CPSC 210 (Introduction to Software Engineering) course. The game requires the user to guess a randomly chosen restaurant’s location by clicking on the map of the associated city, with the ability to use up to of 3 hints. For each second spent, the user’s score drops, with this score dropping further when the user makes an incorrect guess.

Within the app, the gradient of the marker changes based on proximity to the actual location, and users have a choice on the city/type of restaurant that they’d like to play in. Once the user clicks within 50m of the actual location, they “win” and their score is updated to the scoreboard, which holds the highest score and the fewest attempts on record.

Done in pairs, this was my first step into the Android Development world.

The initial stages required us to write JSON parsers for both the Google Maps data and the Yelp data. Following this, we had to work with the Google Maps API to place markers on the map where the user had clicked and to plot the total route between all such points.

The learning curve was quite steep, as we had to learn how to use various APIs, parsers, nested classes and onClickListener classes to complete the game functionality.

Built using: Java, JSON, Yelp Api, Google Maps Api | GitHub
